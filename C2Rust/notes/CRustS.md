# CRustS: C to Rust via Source Transformation

## 基本信息
- **标题**: CRustS: C to Rust via Source Transformation
- **作者**: Yijun Yu 等
- **机构**: Huawei
- **年份**: 2023
- **发表形式**: 技术报告/预印本
- **开源代码**: https://github.com/nickyc975/crusts

## 核心问题
如何通过源代码级别的变换规则，系统性地将C代码转换为更安全的Rust代码？

## 主要贡献
1. **TXL规则集**: 220条源代码转换规则
2. **分类设计**: 198条语义保持规则 + 22条语义近似规则
3. **可验证性**: 规则可审计和定制

## 技术方法

### 基于TXL的变换
- **TXL**: 一种源代码变换语言
- **优势**: 声明式规则，易于理解和验证
- **工作流**: C → (TXL规则) → Rust

### 规则分类

#### 语义保持规则 (198条)
完全保持程序语义的变换：
- 类型映射: `int` → `i32`, `char*` → `String`
- 控制流: `for` → `loop`, `switch` → `match`
- 表达式: 运算符、函数调用等

#### 语义近似规则 (22条)
可能改变行为的变换：
- 指针处理: 原始指针 → 引用
- 内存管理: `malloc/free` → `Box/Vec`
- 全局变量: 简化处理策略

### 实现细节
- 规则可组合应用
- 支持增量变换
- 保留无法变换的代码为unsafe

## 主要结果
- 在测试程序上生成更高比例的safe代码
- 规则可追溯，变换可审计
- 支持人工干预和定制

## 优点
1. **透明性**: 每条规则可独立检验
2. **可定制**: 用户可添加/修改规则
3. **工业适用**: 符合华为的软件工程实践

## 局限性
1. **规则维护**: 220条规则需要持续维护
2. **覆盖性**: 可能遗漏某些C模式
3. **语义近似风险**: 22条近似规则可能引入bug

## 工业视角
- **华为背景**: 大规模C代码库迁移需求
- **合规需求**: 变换过程需要可审计
- **增量采用**: 支持逐步迁移策略

## 评价
**优点**:
- 工业实用性强
